# Executive Protocol: Autonomous Repository Management

This document specifies the autonomous synchronization and development protocol for the LiquidGov project.

## 1. Upstream Synchronization
- **Fetch All**: Periodically fetch all remotes and tags.
- **Submodule Sync**: Recursively update all submodules using the `--remote` flag to track the latest commits.

## 2. Dual-Direction Intelligent Merge Engine
- **Forward Merge**: AI-generated feature branches (prefixed with `jules-` or `main-`) are automatically merged into the `main` branch.
- **Reverse Merge**: The updated `main` branch is merged back into all active feature branches to prevent drift and ensure version consistency.
- **Conflict Handling**: If a merge conflict occurs, the specific branch is skipped and the merge is aborted to maintain stability.

## 3. Versioning & Documentation Governance
- **Automated Versioning**: Every successful protocol execution increments the patch version in `VERSION.md` and `package.json`.
- **Documentation Sync**: `CHANGELOG.md`, `ROADMAP.md`, `TODO.md`, and `HANDOFF.md` are updated to reflect the new state.
- **System Build**: A full system build is executed using `scripts/build.sh` or `build.bat` to ensure code integrity.
- **Session Handoff**: A comprehensive summary of session activities (merges, bumps) is automatically generated in `HANDOFF.md`.
- **Mandatory Standards**: The project must maintain:
  - `VISION.md`
  - `MEMORY.md`
  - `DEPLOY.md`
  - `IDEAS.md`
  - `AGENTS.md`

## 4. Continuous Integration
- The protocol is integrated into GitHub Actions (`autopilot.yml`).
- Commits generated by the protocol use the `[skip ci]` tag to prevent infinite loops.
- All tests must pass before the protocol proceeds with merging.
